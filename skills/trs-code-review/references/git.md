# Git 与提交约定参考

脚手架项目已在 commit 阶段触发团队校验。本文件只记录协作约定。

## 提交粒度

- 一次提交只做一类事情。
- 不把格式化、重构、功能、修 bug 混在一个大提交里。
- 提交前查看 diff，避免带入无关文件。

## Commit Message

推荐 Conventional Commits：

```text
feat: 新增用户列表筛选
fix: 修复表单回显异常
refactor: 拆分列表页表单逻辑
style: 调整按钮间距
```

## MR / PR 说明

建议包含：

- 改动内容
- 影响范围
- 接口变更
- 验证方式
- 风险点和回滚方式
