# 调试与性能参考

Superpowers 已提供系统化调试和验证流程。本文件只保留 TRS 前端项目常见关注点。

## 前端调试关注点

- UI 问题：先确认 props、响应式状态、条件渲染和样式覆盖。
- 请求问题：检查请求参数、响应结构、错误码、loading 关闭时机。
- 状态问题：检查 Pinia 是否重复请求、状态是否被非预期共享或覆盖。
- 样式问题：检查容器尺寸、`min-height: 0`、`min-width: 0`、滚动容器和 z-index。

## chrome-devtools-mcp 使用边界

`chrome-devtools-mcp` 用于真实浏览器证据收集，不替代代码规范、类型检查、单测或构建。

建议使用的场景：

- 页面、组件、样式、交互、接口联调或性能问题。
- 用户反馈“页面打不开、样式错乱、点击没反应、数据不显示、请求失败”等运行时问题。
- 开发完成后需要确认真实页面效果，而不只是确认代码能编译。

可不使用的场景：

- 纯文档、纯类型声明、脚本配置、依赖元信息等无 UI/运行时影响的改动。
- 只改注释、命名或被 lint/format 明确覆盖的机械调整。

验证时至少记录：

- 访问的页面路径和主要操作。
- Console 是否有新增错误。
- 关键 Network 请求的参数、状态码和响应结构。
- 页面主要布局、交互状态或截图结论。

## 性能关注点

- 页面加载：路由懒加载、资源体积、接口并发、首屏骨架。
- 运行时：长列表、频繁 watch、重复计算、图表 resize、无节制轮询。
- 大屏项目：图表实例销毁、resize 防抖、定时器清理、数据刷新频率。
- 构建体积：按需引入、动态 import、可视化分析。
